Getting Started

The Discount Customer API provides access to events, categories, and location data. The base URL for all API requests is:

http://localhost:8000/{locale}/api/v3/
Locale Support: The API supports multiple locales. Replace {locale} with your desired language code (e.g., en, tr).

Authentication

Authentication endpoints for user registration, login, and profile management.

POST Login

http://localhost:8000/en/api/v3/auth/login

Description: Authenticate a user and receive a session token.

Request Body:

{ "email": "customer@gmail.com", "password": "123" }

Response:

Returns a session token stored in cookies (customer_session_token).

POST Register

http://localhost:8000/en/api/v3/auth/register

Description: Create a new customer account.

Request Body:

{ "email": "customer@gmail.com", "password": "123", "firstname": "nika", "lastname": "shabashvili", "country": "geo", "mobile": "577123" }
Parameter Type Required Description
email string Yes User's email address
password string Yes User's password
firstname string Yes User's first name
lastname string Yes User's last name
country string Yes Country code
mobile string Yes Mobile phone number

GET Profile

http://localhost:8000/en/api/v3/auth/profile

Description: Retrieve the authenticated user's profile information.

Authentication Required: This endpoint requires a valid session token.

POST Logout

http://localhost:8000/en/api/v3/auth/logout

Description: End the user's session and invalidate the session token.

Events

Comprehensive event discovery and management endpoints.

GET Event Feed

http://localhost:8000/tr/api/v3/event/feed

Description: Retrieve the main feed of events.

No Authentication Required

GET Event Details

http://localhost:8000/en/api/v3/event/details/{id}

Description: Retrieve detailed information about a specific event.

Path Parameters:

Parameter Type Description
id integer The unique identifier of the event

Example:

GET http://localhost:8000/en/api/v3/event/details/1

GET Search Events

http://localhost:8000/en/api/v3/event/search?q={query}

Description: Search for events by keyword.

Query Parameters:

Parameter Type Required Description
q string Yes Search query string

Example:

GET http://localhost:8000/en/api/v3/event/search?q=concert

GET Discounted Events

http://localhost:8000/en/api/v3/event/feed/discounted

Description: Retrieve events that currently have discounts available.

No Authentication Required

GET Featured Events

http://localhost:8000/en/api/v3/event/feed/featured

Description: Retrieve featured events curated by the platform.

No Authentication Required

GET Popular Events

http://localhost:8000/en/api/v3/event/feed/popular

Description: Retrieve events sorted by popularity.

No Authentication Required

POST Price Calculate

http://localhost:8000/en/api/v3/event/price-calculate

Description: Calculate the total price for an event based on the number of attendees.

Request Body:

{ "event_id": 1, "people_count": 10 }
Parameter Type Required Description
event_id integer Yes The ID of the event
people_count integer Yes Number of people attending

Response:

Returns the calculated total price including any applicable discounts.

Authentication Flow

The API uses session-based authentication with cookies:

  1. Register a new account using the /auth/register endpoint
  2. Login with credentials using the /auth/login endpoint
  3. The server sets a customer_session_token cookie (valid for 48 hours)
  4. Include this cookie in subsequent requests to authenticated endpoints
  5. Use /auth/logout to end the session
Cookie Details:
  • customer_session_token: Session authentication token
  • csrftoken: CSRF protection token for POST requests

Error Handling

The API uses standard HTTP status codes to indicate success or failure:

Status Code Description
200 Success - Request completed successfully
400 Bad Request - Invalid parameters or request body
401 Unauthorized - Authentication required or invalid credentials
404 Not Found - Resource does not exist
500 Server Error - Internal server error occurred
